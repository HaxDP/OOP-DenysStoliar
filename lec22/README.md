# Лекція №22

## опис
цей документ супроводжує файл лекції №22 і стисло пояснює, що саме вивчається в темі файлового I/O та серіалізації даних у форматах JSON і XML. акцент зроблено на практичних підходах, які можна одразу застосовувати в навчальних та робочих проєктах на платформі .NET.

## мета лекції
основна мета матеріалу полягає у формуванні цілісного розуміння роботи з файлами: від простих операцій читання і запису до контрольованої роботи з потоками та асинхронними API. окремо показано, як переводити об’єкти у формат JSON або XML і відновлювати їх назад у програмні сутності без втрати структури даних.

## що покриває матеріал
лекція розглядає базовий інструментарій простору імен System.IO, зокрема клас File для швидких операцій і StreamReader або StreamWriter для сценаріїв, де потрібен контроль за ресурсами та робота з великими обсягами даних. також пояснюється, як правильно працювати з директоріями, перевіряти існування файлів і уникати типових помилок доступу до файлової системи.

у частині серіалізації наведено підхід через System.Text.Json, параметри форматування, правила іменування властивостей, ігнорування null-значень та асинхронний запис або читання через FileStream. для XML показано використання XmlSerializer, керування іменами елементів і роботу з колекціями об’єктів.

## практична цінність
матеріал орієнтований на реальні задачі: збереження налаштувань програми, обмін даними між сервісами, формування звітів, логування подій та інтеграцію з зовнішніми системами. окрема увага приділена обробці винятків, щоб програма коректно реагувала на відсутність файлу, проблеми доступу або помилки парсингу JSON чи XML.

## JSON та XML у порівнянні
| критерій | JSON | XML |
|   ----   | ---- | --- |
| розмір | компактний синтаксис | більш об’ємний формат |
| читабельність | висока для прикладних задач | середня, але структурно виразна |
| швидкість обробки | зазвичай вища | зазвичай нижча |
| формальна валідація | JSON Schema (опційно) | XSD (вбудований підхід) |
| типові сценарії | REST API, веб і мобільні застосунки | SOAP, enterprise-інтеграції, legacy-системи |

## висновок/висновки
лекція №22 формує практичну базу для безпечної роботи з файлами та коректної серіалізації даних у .NET. після опрацювання матеріалу можна впевнено обирати між JSON і XML залежно від контексту, грамотно організовувати I/O-операції та будувати стійкі до помилок прикладні рішення.
