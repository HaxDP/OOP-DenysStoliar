# Лекції 19-20

## Composite

Патерн Composite дозволяє будувати ієрархію "частина-ціле" так, щоб клієнт однаково працював і з окремим елементом, і з групою елементів. Це досягається спільним інтерфейсом Component, де Leaf реалізує кінцеву поведінку, а Composite зберігає дочірні елементи та делегує їм виклики. У результаті дерево обробляється як єдина колекція без зайвих перевірок типів у клієнтському коді.

### Де Composite найбільш доцільний

У файловій системі файл поводиться як Leaf, а папка як Composite, тому операції на кшталт підрахунку розміру працюють уніфіковано. У графічному редакторі окрема фігура є Leaf, а група фігур є Composite, тому переміщення чи рендер викликаються через один інтерфейс. Схожа модель добре працює в організаційній структурі компанії, де працівник є Leaf, а відділ є Composite.

### Переваги Composite

Composite уніфікує API, зменшує кількість умов у клієнтській логіці, добре масштабується і природно підтримує рекурсивну обробку дерева.

### Недоліки Composite

Недоліки теж є: інколи важко жорстко обмежити склад дочірніх елементів, інтерфейс може стати занадто загальним, а валідація структури дерева ускладнюється.

## Adapter, Facade, Proxy

### Adapter

Adapter застосовується, коли треба інтегрувати клас із несумісним API без зміни стороннього коду. Типові приклади: адаптація старого платіжного API до нового контракту, з'єднання XML-бібліотеки з JSON-орієнтованим клієнтом, або підключення зовнішнього SDK до внутрішнього інтерфейсу домену.

### Facade

Facade доречний, коли підсистема складна, а клієнту потрібна проста точка входу. Наприклад, фасад для домашнього кінотеатру, платіжного процесу або ініціалізації інфраструктури. Основний ефект полягає у зменшенні складності та прихованні деталей взаємодії класів.

### Proxy

Proxy варто використовувати, коли треба контролювати доступ до об'єкта або додати кешування, логування чи ледаче завантаження без зміни реальної реалізації. Приклади: virtual proxy для великих зображень, protection proxy для доступу за ролями, remote proxy для віддаленого сервісу.

### Ключова відмінність між Adapter, Facade, Proxy

Adapter змінює інтерфейс для сумісності, Facade спрощує взаємодію зі складною підсистемою, а Proxy додає контроль доступу та додаткову поведінку навколо реального об'єкта.

## Висновок

Composite корисний, коли домен природно має деревоподібну структуру, а Adapter, Facade і Proxy закривають різні задачі інтеграції, спрощення та контролю доступу.
