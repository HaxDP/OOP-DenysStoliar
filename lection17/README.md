# Factory vs Abstract Factory: вибір. Лекція №17 + ДЗ
**factory method і abstract factory** *- це породжувальні патерни проєктування, які допомагають створювати об'єкти без жорстокої прив'язки до конкретних класів*

---

## різниця між патернами
**factory method**


*дозволяє підкласам вирішувати, який саме об'єкт створювати. є один метод-фабрика, що повертає об'єкт певного типу*


**abstract factory**

*надає інтерфейс для створення цілої групи пов'язаних об'єктів, не вказуючи їх конкретні класи*

---

## використання
**factory method**

- *потрібно створювати різні варіанти одного об'єкта*
- *тип об'єкта відомий лише під час виконання*

*приклад:* батько, мати, дочка

**abstract factory**

- *потрібно створювати групи пов'язаних об'єктів*
- *важливо, щоб вони були суміжні між собою*

*приклад:* GUI для різних ОС

---

## переваги і недоліки
**factory method**

$переваги:$
- *простий у реалізації*
- *гнучкість у створенні об'єктів*
- *менше залежностей від конкретних класів*

$недоліки:$
- *багато підкласів*
- *не підходить для створення складних груп об'єктів*

**abstract factory**

$переваги:$
- *гарантує сумісність об'єктів*
- *легко міняти цілі сім'ї продуктів*
- *добре підходить для масштабних систем*

$недоліки:$
- *складніша реалізація*
- *важко додавати новий тип продукту*

---

## висновок
- *якщо треба окремі об'єкти то використовувати* **factory method**
- *якщо треба набори пов'язаних об'єктів то використовувати* **abstract factory**
