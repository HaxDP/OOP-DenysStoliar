# самостійна робота №16

консольна програма на c++, що демонструє рефакторинг коду кошика покупок. показує перехід від монолітного класу до архітектури з розділенням відповідальностей через dependency injection

## що робить
- порівнює два підходи: погано спроектований клас та рефакторену версію
- додає та видаляє товари з кошика
- розраховує загальну вартість кошика
- зберігає стан кошика
- каталог містить телефон який коштує 300 та ноутбук вартістю 1000

## архітектура
- BadShoppingCart: все в одному класі (каталог, логіка, розрахунок, збереження)
- ShoppingCartService: розділена архітектура з інтерфейсами:
  - IProductCatalog: управління каталогом товарів
  - ICartCalculator: розрахунок вартості
  - ICartStorage: збереження кошика
- демонструє принципи SOLID та dependency injection
